## Codebase Patterns

### CIDÔćĺUnicode Table Pattern
- Tables are stored as sorted static arrays of `(CID, Unicode)` tuples in dedicated modules (e.g., `adobe_gb1_ucs2.rs`)
- Lookup uses `binary_search_by_key` for O(log n) performance
- Tables are generated from Adobe's open-source `cmap-resources` repo (`cid2code.txt` files)
- Column used: `UniXX-UCS2` (e.g., `UniGB-UCS2`, `UniCNS-UCS2`, `UniKS-UCS2`)
- For multi-value entries, prefer CJK Unified Ideograph range (0x4E00-0x9FFF)
- Wire into interpreter.rs via `match si.ordering.as_str()` in the CIDÔćĺUnicode fallback chain (step 4)

### CIDÔćĺUnicode Fallback Chain (interpreter.rs)
1. ToUnicode CMap lookup
2. CidToGid + CFF glyph name
3. Pre-Unicode CMap decoding (encoding_rs) ÔÇö includes EUC-JP, Shift-JIS, GBK, Big5, EUC-KR, and raw JIS X 0208
4. Adobe CIDÔćĺUnicode table (Japan1, GB1, CNS1, Korea1)
5. Fallback: Identity-H/V + Identity ordering Ôćĺ `char::from_u32(cid)`; other CID fonts Ôćĺ `(cid:N)`; simple fonts Ôćĺ `char::from_u32`

### CJK Encoding CMap Name Mapping
- `GBK-EUC-H/V`, `GBKp-EUC-H/V`, etc. Ôćĺ GBK (encoding_rs::GBK)
- `ETen-B5-H/V`, `B5pc-H/V`, etc. Ôćĺ Big5 (encoding_rs::BIG5)
- `90ms-RKSJ-H/V`, etc. Ôćĺ Shift-JIS (encoding_rs::SHIFT_JIS)
- `EUC-H/V` Ôćĺ EUC-JP (encoding_rs::EUC_JP)
- `H`, `V` Ôćĺ Raw JIS X 0208 (tagged as encoding_rs::ISO_2022_JP, decoded via JISÔćĺEUC-JP conversion)
- `KSC-EUC-H/V`, etc. Ôćĺ EUC-KR (encoding_rs::EUC_KR)
- JISÔćĺUnicode: add 0x80 to each byte Ôćĺ EUC-JP Ôćĺ encoding_rs decode
