## Codebase Patterns

### Word grouping direction handling
- WordExtractor merges horizontal (Ltr + Rtl) and vertical (Ttb + Btt) chars separately
- Horizontal chars always use spatial LTR sorting (ascending x0) matching Python pdfplumber
- Vertical chars always use spatial TTB sorting (ascending top) matching Python pdfplumber
- Per-char direction from TRM determines horizontal vs vertical grouping, NOT reading order
- Python pdfplumber separates by `upright` flag: upright=true → horizontal, upright=false → vertical

### Rotation and direction mapping
- 0° rotation: direction=Ltr, upright=true
- 90° rotation: direction=Ttb (|b|>|a|, b<0), upright=false
- 180° rotation: direction=Rtl (a<0), upright=true
- 270° rotation: direction=Btt (|b|>|a|, b>0), upright=false
- Direction determined in char_extraction.rs from Text Rendering Matrix (TRM)
