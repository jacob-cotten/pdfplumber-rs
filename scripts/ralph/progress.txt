## Codebase Patterns

### Content Stream Error Recovery
- `tokenize()` is strict (returns `Err` on malformed tokens)
- `tokenize_lenient()` recovers by saving position before each token, catching errors, resetting position to `saved_pos + 1`, and continuing
- Failed parse functions (e.g., `parse_literal_string`) may advance `pos` to EOF before returning error; always reset to `saved_pos + 1` for robust recovery
- Operand stack is cleared on error to prevent partial state corruption
- Interpreter uses `tokenize_lenient()` and emits `ExtractWarning` with `MalformedObject` code for each recovery
- `Do` operator errors are caught except for recursion depth limits (resource limits must propagate)
